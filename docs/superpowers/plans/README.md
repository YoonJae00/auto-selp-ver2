# Implementation Plans

Last updated: 2026-05-20

Files in this directory are historical planning artifacts. They explain why a feature was designed a certain way and can still be useful when continuing nearby work, but their checkbox state is not the project source of truth.

Use these sources for current status:

- `../../../TODO.md` for the active backlog snapshot.
- `../current_state.md` for the current implementation snapshot.
- GitHub open issues for active tracked work.

When a plan appears to conflict with current code, prefer the current code and the two status documents above. Update this directory only when writing a new implementation plan or when a plan is explicitly revived for active work.
