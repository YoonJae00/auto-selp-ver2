# [Design Spec] 상품 가공 탭 UX/UI 개선 (선택 상품 가공 버튼, 상단 페이징, 상품명 검색)

본 문서는 상품 가공 페이지의 핵심 액션 동선을 최적화하고, 대시보드의 사용성을 극대화하기 위한 UI/UX 개선안 설계 스펙입니다.

## 1. 개선 배경 및 목표

* **현재 문제점**:
  1. **액션 버튼 이격**: '선택 상품 가공' 버튼이 페이지 우측 최상단 헤더에 있어, 테이블 좌측에서 상품을 체크한 뒤 마우스를 우측 상단 끝까지 멀리 이동해야 하는 동선 낭비가 발생합니다.
  2. **페이징 제어 불편**: 상품 데이터가 50개 단위로 세로로 길게 나열되는 동안 '이전/다음' 버튼이 하단에만 존재하여, 필터 조작 후 페이지를 이동하려면 매번 스크롤을 맨 밑으로 내려야 하는 불편함이 있습니다.
  3. **검색 기능 부재**: DB 내에 대량의 상품이 있을 때 특정 상품명을 검색하여 선별적으로 가공하고 싶어도 상품명 검색창이 존재하지 않아 상품을 찾기 어렵습니다.

* **개선 목표**:
  * 테이블 근처에 직관적인 액션 실행 위치 제공 (상시 테이블 툴바 배치)
  * 다이나믹 스크롤 대응용 하단 플로팅 액션 바 도입 (Premium Apple-style UX)
  * 상단 필터바 내에 '상품명 검색' 및 '이전/다음 페이징' 통합

## 2. 세부 UI/UX 변경 사항

### 2.1. 선택 상품 가공 버튼 개선 (이원화 배치)

#### [개선 1] 테이블 툴바(`tableToolbar`) 내 배치
* **위치**: 테이블 바로 위의 제목 우측 영역에 "현재 페이지 전체 선택" 체크박스와 나란히 배치합니다.
* **디자인**: 파란색 주 액션 버튼 스타일 (`PillButton` variant="primary").
* **상태**: 
  - 선택된 상품이 0개일 때: `disabled` (연한 회색 배경, 클릭 불가)
  - 선택된 상품이 1개 이상일 때: `active` 상태로 변경되어 활성화.

#### [개선 2] 하단 플로팅 액션 바(`floatingActionBar`) 도입
* **조건**: 선택된 상품(`selectedIds.size`)이 **1개 이상일 때만** 하단에 고정 노출됩니다.
* **비주얼**:
  - Glassmorphic 스타일 적용 (`backdrop-filter: blur(16px)`, `background: rgba(255, 255, 255, 0.85)`)
  - 은은하고 고급스러운 그림자 효과 (`box-shadow: 0 12px 40px rgba(0, 0, 0, 0.12)`)
  - 나타날 때 부드럽게 위로 솟아오르는 슬라이드 애니메이션 (`slideUp`)
* **구성**:
  - 좌측: `✨ N개의 상품이 선택되었습니다.` (볼드 처리된 강조 텍스트)
  - 우측: `[선택 상품 가공]` (Primary 버튼), `[선택 취소]` (Secondary/Link 버튼)

---

### 2.2. 필터바(`filterBar`) 개편

필터바의 가로 구조를 Flex 양방향 레이아웃(`justify-content: space-between`)으로 재설계합니다.

#### [좌측] 검색 및 필터 그룹
1. **[NEW] 상품명 검색창 (`searchGroup`)**:
   - `input` 태그와 함께 내부에 은은한 돋보기 아이콘 및 검색 텍스트 배치.
   - 우측에 입력 내용이 있을 때 즉시 지울 수 있는 `(X)` 초기화 버튼 배치.
   - Enter 키 입력 시 또는 검색 버튼 클릭 시 API 호출을 통해 즉시 DB 검색 연동.
2. **[기존] 가공 상태 필터**: 상태 필터 드롭다운.
3. **[기존] 정렬 필터**: 정렬 드롭다운.
4. **[기존] 가공 완료만 보기**: 토글 체크박스.

#### [우측] 상단 페이징 제어기 (`topPagination`)
* **위치**: 필터바 우측 정렬
* **구성**: `[이전]` (아이콘 혹은 버튼) `현재페이지 / 전체페이지` `[다음]`
* **기능**: 하단 페이징 제어기와 완벽하게 동기화되어 작동.

## 3. 기술적 구현 설계 (Next.js & CSS)

### 3.1. API 호출 확장
* 기존 `fetchProducts` 호출 시 `searchQuery` 상태값을 쿼리 매개변수로 전달합니다.
* backend `/api/processor/products` 엔드포인트는 이미 `search` 파라미터를 지원하고 있으므로, 프론트엔드에서 `search` 키에 검색어를 바인딩하여 전송합니다.

```typescript
const params = new URLSearchParams({
  page: String(page),
  size: String(pageSize),
  wholesale_site_id: activeSiteId,
});
if (searchQuery.trim()) {
  params.append('search', searchQuery.trim());
}
```

### 3.2. CSS 스타일 추가 및 개선

* `.filterBar`에 `display: flex; justify-content: space-between; align-items: center;` 구조를 잡습니다.
* `.floatingActionBar`에 대한 CSS와 `@keyframes slideUp` 트랜지션을 정의합니다.
* 모바일 환경(`@media`) 반응형 대응 시 플로팅 바 및 필터바 레이아웃이 자연스럽게 떨어지도록 조절합니다.

## 4. 검증 계획 (Verification Plan)

### 수동 검증 사항
1. **버튼 위치 검증**: 
   - 페이지 최상단 헤더 우측의 '선택 상품 가공' 버튼이 삭제되었는지 확인.
   - 테이블 상단 툴바 우측에 '선택 상품 가공' 버튼이 정상 노출되며 선택 상품 개수가 실시간 연동되는지 확인.
2. **플로팅 바 검증**:
   - 상품을 1개 체크하면 하단에 플로팅 바가 부드럽게 나타나는지 확인.
   - 모든 체크를 해제하면 플로팅 바가 즉시 사라지는지 확인.
   - 플로팅 바의 '선택 상품 가공' 및 '선택 취소' 버튼이 정상 작동하는지 확인.
3. **검색창 연동 검증**:
   - 상품명을 검색창에 입력하고 Enter를 치거나 돋보기를 눌렀을 때, 도매처 내의 상품명이 정상적으로 필터링되어 출력되는지 확인.
   - 입력값 제거 후 초기화 시 원래 상품 목록으로 올바르게 복귀하는지 확인.
4. **상/하단 이중 페이징 검증**:
   - 상단 페이징의 '이전/다음'을 눌렀을 때 하단 페이징 수치와 테이블 데이터가 일치하며 올바르게 전이되는지 확인.
