# 상세 설계서: Auto-Selp 이커머스 자동화 플랫폼

## 1. 개요 (Overview)
Auto-Selp는 셀러들이 상품 데이터를 최적화하고, 매출 성과를 추적하며, 향후 상품 등록부터 CS까지 전체 판매 라이프사이클을 관리할 수 있도록 돕는 종합 SaaS 기반 이커머스 자동화 허브입니다. 이 시스템은 확장성과 신규 기능 추가의 용이성을 확보하기 위해 마이크로서비스 아키텍처(MSA)를 기반으로 구축됩니다.

## 2. 아키텍처: 마이크로서비스 (MSA)
시스템은 Docker Compose를 통해 오케스트레이션되는 독립적인 서비스들로 구성됩니다.

### 2.1 주요 서비스
- **API 게이트웨이 (Nginx)**: 모든 클라이언트 요청의 진입점입니다. 라우팅, SSL 종료 및 기본적인 보안을 담당합니다.
- **인증 서비스 (Auth Service, FastAPI)**:
    - 사용자 등록, 로그인(JWT), 역할 기반 권한 제어(관리자/소비자)를 관리합니다.
    - 네이버, 쿠팡 등 플랫폼 API 키를 AES-256 알고리즘으로 암호화하여 안전하게 저장합니다.
- **대시보드 서비스 (Dashboard Service, FastAPI)**:
    - 실시간 및 과거 통계 데이터(일일 판매량, 주간 추세, 매출 차트 등)를 제공합니다.
    - 메인 페이지의 빠른 응답 속도를 위해 읽기 작업에 최적화되어 있습니다.
- **상품 가공 서비스 (Product Processor, FastAPI)**:
    - 상품명 정제, 키워드 추출, 카테고리 매칭의 핵심 로직을 담당합니다.
    - 무거운 작업은 백그라운드 워커로 위임합니다.
- **워커 (Worker, Celery)**:
    - 엑셀 파싱, LLM 호출(Gemini), 플랫폼 API 요청 등 시간이 오래 걸리는 작업을 실행합니다.
- **데이터베이스 (PostgreSQL)**: 서비스별로 스키마를 분리하여 운영하는 공통 데이터베이스 인스턴스입니다.
- **메시지 브로커 (Redis)**: Celery를 위한 작업 큐 역할을 하며, 대시보드용 캐시로도 활용됩니다.

## 3. 핵심 기능 로직 (상품 가공)

### 3.1 파이프라인 단계
1. **정제(Refinement)**: LLM을 사용하여 상품명에서 브랜드, 특수문자, 중복 단어를 제거하고 가독성을 높입니다.
2. **키워드 선별(Keyword Curation)**:
    - 1단계: 네이버 검색광고 API 및 쿠팡 웹 어댑터를 통한 시드 키워드 수집.
    - 2단계: 경쟁도, 길이, 불용어 리스트를 기반으로 필터링.
    - 3단계: 로컬 블랙리스트 및 LLM 문맥 검사를 통한 상표권 검증.
3. **카테고리 매칭(Category Mapping)**:
    - 네이버: 쇼핑 검색 API 경로 매칭 후 로컬 매핑 파일 대조.
    - 쿠팡: 쿠팡 카테고리 예측 API 직접 호출.

## 4. 데이터 전략
- **보안**: 사용자 API 키는 저장 시 암호화됩니다. 서비스 간 및 클라이언트-서비스 간 통신에는 JWT 인증이 사용됩니다.
- **병렬 처리**: Celery 워커가 엑셀의 각 행을 병렬로 처리하여 시스템 병목 현상을 방지합니다.
- **확장성**: 각 서비스가 컨테이너화되어 있어, 가공 서비스나 대시보드 서비스만 독립적으로 수평 확장(Scaling)할 수 있습니다.

## 5. UI/UX 로드맵 (SaaS)
- **메인 대시보드**: 매출 데이터를 시각화한 차트 제공.
- **가공 페이지**: 엑셀 업로드(드래그 앤 드롭), 비동기 작업의 실시간 상태 추적.
- **설정 페이지**: 사용자 프로필 및 플랫폼 API 자격 증명 관리.

## 6. 기술 스택 (Tech Stack)
- **백엔드**: Python 3.12+, FastAPI, Celery, SQLAlchemy (Async).
- **프론트엔드**: React/Next.js (TypeScript).
- **데이터베이스**: PostgreSQL 16.
- **인프라**: Docker, Docker Compose, Redis.
- **AI**: Gemini 2.0 Flash API.
