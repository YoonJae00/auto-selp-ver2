# Auto-Selp: 핵심 상품 가공 로직 가이드 (Core Logic)

이 문서는 Auto-Selp 시스템의 핵심 엔진인 상품 가공 파이프라인의 알고리즘과 구현 세부 사항을 설명합니다. 새로운 개발자가 로직을 재구현하거나 확장할 때 참고하십시오.

---

## 1. 상품 가공 파이프라인 개요
시스템은 **파이프라인 아키텍처**를 따르며, 각 단계(Stage)는 이전 단계의 출력물을 입력으로 받아 데이터를 점진적으로 풍성하게 만듭니다.

**순서**: `원본 상품명` → `정제된 상품명` → `키워드 추출` → `카테고리 매칭`

---

## 2. [Stage 1] 상품명 정제 (Product Name Refinement)
LLM을 사용하여 검색 엔진이 선호하는 깔끔한 상품명으로 가공합니다.

### 2.1 주요 로직
1.  **브랜드/특수문자 제거**: 상품 본연의 명칭만 남깁니다.
2.  **수량 표준화**: 
    - `10p`, `10pcs`, `10입` → `10개`로 통일.
    - 단독 `1p`, `1개` 등 의미 없는 수량 키워드는 제거.
3.  **LLM 재시도 전략 (Robustness)**:
    - 결과가 JSON이거나 빈 값인 경우를 대비해 3가지 다른 프롬프트로 최대 3회 재시도합니다.
    - 정제 실패 시 원본 상품명을 유지하여 데이터 손실을 방지합니다.

---

## 3. [Stage 2] 키워드 큐레이션 (Keyword Curation)
이 시스템의 가장 복잡한 부분으로, **3-Phase 워크플로우**를 따릅니다.

### Phase 1: 다각도 시드(Seed) 수집
- **Round 1**: 원본/정제 상품명으로 네이버 광고 API(`keywordstool`) 및 쿠팡 연관 검색어 호출.
- **Round 2**: LLM을 통해 상품명의 동의어(예: "무선 이어폰" → "블루투스 이어셋")를 2~3개 생성하여 추가 API 호출.
- **결과**: 수백 개의 잠재 키워드 리스트 확보.

### Phase 2: 데이터 기반 필터링 (알고리즘)
수집된 키워드 중 다음 기준에 따라 '품질 점수'를 계산하고 필터링합니다.
1.  **경쟁도 필터**: `compIdx == "높음"` 인 키워드는 대형 브랜드의 영역이므로 제거.
2.  **불용어(Stop Words) 제거**: `랜덤`, `박스채`, `배송비` 등 판매와 무관한 키워드 제거.
3.  **롱테일 보너스**: 
    - 단어 수가 2개 이상일 때 가점.
    - 3개 이상(예: "여름용 린넨 바지")일 때 최고 가점.
4.  **수량 패턴 제거**: `2개`, `3세트` 등 규격 키워드는 정규표현식으로 제거.

### Phase 3: 상표권 및 AI 검증
1.  https://github.com/nuri428/mcp_kipris 를 확인하고 mcp를 설치하세요.

---

## 4. [Stage 3] 카테고리 매칭 (Category Matching)

### 4.1 네이버 쇼핑 카테고리
1.  **API 검색**: `shop.json` API로 상품명을 검색하여 상위 1개 아이템의 카테고리 경로(`cat1>cat2>cat3>cat4`)를 가져옵니다.
2.  **로컬 매핑**: `data/naver_category_mapping.xls` 파일에서 해당 경로를 찾아 5~8자리 카테고리 번호(ID)를 획득합니다.
3.  **부분 일치 로직**:
    - 경로 전체가 일치하지 않을 경우, 가장 하위 단계인 `세분류` 명칭만으로 매핑 테이블을 다시 검색합니다.

### 4.2 쿠팡 카테고리
- 쿠팡은 별도의 매핑 테이블 없이 **Predict API**를 직접 활용합니다.
- **입력 데이터**: `productName`, `brand`, `attributes`(속성)
- **핵심 파라미터**: `autoCategorizationPredictionResultType`이 `SUCCESS`인 경우에만 `predictedCategoryId`를 사용합니다.

---

## 5. 구현 시 주의사항 (Developer Tips)

### API 인증 보안
- **네이버 광고 API**: 매 요청마다 `Timestamp`를 포함한 HMAC-SHA256 시그니처를 생성해야 합니다. (시각 동기화 중요)
- **쿠팡 API**: `CEA` 알고리즘 기반의 복잡한 헤더 생성이 필요하며, `Secret Key` 노출에 주의하십시오.

### 에러 핸들링
- 외부 API(네이버, 쿠팡, LLM)는 호출 빈도 제한(Rate Limit)이 엄격합니다. 
- **Backoff 전략**: 429(Too Many Requests) 에러 발생 시 대기 시간을 늘려가며 재시도하는 로직이 필수입니다.
- **Fallback**: API 실패 시 빈 값을 반환하기보다 "확인필요"와 같은 플래그를 남겨 사용자가 수동으로 개입할 수 있게 합니다.
