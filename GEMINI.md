# Auto-Selp Project Instructions

## Progress Tracking Rule
- **Mandatory Documentation Update:** Every time a task or a part of a feature is completed, you MUST update the relevant documentation (e.g., `TODO.md`, `current_state.md`, or specific status files) to reflect the completion status.
- **Verification:** Before claiming completion, ensure the code is implemented and, where possible, verified.

## Tech Stack & Design
- **Frontend:** Next.js (App Router), TypeScript, Vanilla CSS.
- **Backend:** FastAPI (Python), PostgreSQL, Redis.
- **Design:** Follow Apple-inspired guidelines in `design.md`.

## Cleanup Rules
- **Temporary Files Cleanup:** When you finish your development work, you MUST ask the user: "작업이 종료되었습니다. 생성된 임시 파일(예: 테스트 스크린샷, 디버그용 스크립트 등)을 삭제할까요?" (The work is finished. Should I delete the generated temporary files (e.g. test screenshots, debugging scripts, etc.)?) before wrapping up.
